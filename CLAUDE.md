## Project Overview

This is a NixOS system configuration using Flakes with Home Manager integration. The configuration manages both system-level settings and user-level home directory configuration.

## Key Commands

### Apply Configuration
```bash
# Apply the full system configuration (requires sudo)
sudo nixos-rebuild switch --flake .#nixos

# Or use the convenience script (uses Tsinghua mirror for faster builds)
./apply.sh
```

### Common Nix Commands
```bash
# Check flake for errors
nix flake check

# Update flake inputs
nix flake update

# View flake outputs
nix flake show

# Build without switching (dry-run)
nixos-rebuild build --flake .#nixos
```

## Architecture

### File Structure
- `flake.nix` - Flake entry point; defines inputs (nixpkgs, home-manager) and the NixOS configuration
- `configuration.nix` - System-level NixOS configuration (bootloader, networking, users, system packages, services)
- `home.nix` - Home Manager user configuration (user packages, git, zsh, tmux, starship)
- `hardware-configuration.nix` - Auto-generated by `nixos-generate-config`; do not edit manually
- `apply.sh` - Convenience script for applying configuration with mirror

### Home Manager Integration
Home Manager is integrated as a NixOS module (not standalone), configured with:
- `home-manager.useGlobalPkgs = true` - Uses system nixpkgs
- `home-manager.useUserPackages = true` - Installs packages to user profile
- User "one" is defined in both `configuration.nix` (system user) and `home.nix` (home config)

### Important Details
- Uses nixos-25.11 channel
- Substituters configured for Chinese mirrors (USTC, Tsinghua) for faster builds
- User shell is zsh with starship prompt, autosuggestions, and syntax highlighting
- Docker is enabled and starts on boot
- systemd-boot bootloader (UEFI)
